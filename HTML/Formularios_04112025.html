Formularios en HTML5

Los formularios en HTML, son elementos que nos van a dotar las páginas web,
de cierta funcionalidad y dinamismo.
Exactamente, nos van a propocionar y permitir recoger información de los
usuarios, con el fin de poder gestiornar posteriormente dicha información
con por ejemplo bases de datos u otras webs.
Los formularios nos permiten por lo tanto dos tipos de funcionalidades
principales:
  • Recoger información del usuario
  • Enviar esos datos que se han guardado, para procesarlo o realizar
    cualquier tipo de operación que se requiera con ellos.

Los principales objetivos que debemos tener en cuenta a la hora de diseñar un
formulario en HTML, son las siguientes:
  • El formulario tiene que ser lo más sencillo posible a la hora de
    insertar los datos por parte del usuario.
  • La experiencia del usuario / usabilidad (UX), tiene que ser lo más
    agradable posible.
  • Los datos introducidos en los formularios, tienen que tener un formato
    predecible y esperado. Por ejemplo: si vamos a introducir el nombre
    y apellido de un usuario, indicar de manera visual y coherente dónde se
    tiene que introducir... No hacerlo justo en el botón de enviar!!!!
  • Reducir los errores al introducir los datos en la medida de lo posible.
    Y sobre todo, comunicar de manera clara y sencilla al usuario dónde
    está dando el error y que tipo de información es la que se desea gestionar.

Etiquetas básicas para la creación de un Formulario en HTML

Para crear un formulario en HTML5, utilizaremos la etiqueta base form
con apertura y cierre.
A parte de utilizar la etiqueta form que establece que vamos a utilizar un
formulario, tenemos que tener en cuenta una serie de propiedades y Etiquetas
que vamos a utilizar:
  • action: Indica la URL o BBDD o web o correo... (Dirección Backend)
    a la que queremos enviar la información de nuestro formulario.
  • method:El método puede ser get o post.
    Get se utiliza cuando queremos mostrar la información que se envia
    desde nuestro formulario a traves del protocolo HTTP/HTTPS de nuestra web.
    Post, se utiliza cuando queremos ocultar información de nuestro formulario
    a través del protocolo HTTP/HTTPS
  • name: se utiliza para dar un nombre al formulario
  • target: se utiliza para abrir un formulario en otra pestaña
  • entype: sirve para codificar información del envio del formulario.
    Es lo que se podría llamar un token
  • autocomplete: Tiene una función on / off que sirve para activar
    y desactivar el autocompletado de nuestro formulario web.

Tipos de Datos que podemos gestionar en un Formulario HTML5

En un formulario,podemos gestionar cualquier tipo de información que se quiera/requiera.
Pero lo más utilizados o importantes son los siguientes:
  • Información de texto: Nombres, apellidos, direcciones físicas, etc.
    Para ello, utilizaremos las etiquetas input y textarea
    (input type = text)
  • Números o cantidades: Edades, precios, cuantías, etc. Utilizaremos el elemento
    input, pero del tipo numérico (input type = number)
  • Fechas u horas: Fechas de nacimiento, calendarios, etc. Utilizaremos el elemento
    input type date, input type week, etc.
  • Opciones de verdadero o falso: input type checkbox
  • Opciones únicas: podemos elegir entre entre 2 o más opciones. type radio o select
  • Opciones únicas simples: donde utilizaremos select multiple, input checkbox, etc.
  • Opciones abiertas: datalist
  • Seleccionar colores: input type color
  • Seleccionar archivos: input type file

Campos de Entrada de Información Numérica

Para poder obtener información númerica en nuestros formularios, podemos utilizar
los campos númericos. Exactamente, estos dos tipos de campos:
  • input type = number: Con este tipo de input, podemos indicar el valor de
    entrada numérica que queremos introducir en nuestro formulario.
  • Campo de rango: input type = range
Los atributos que van asociados a este tipo de input, son:
  • min: establece un valor mínimo de rango / número que queremos introducir
  • max: establece un valor máximo de rango / número que queremos introducir
  • step: establece la varión de números para pasar de un valor a otro.

Campos de entrada numéricos: fecha y hora

Dentro de los formularios, podemos utilizar un tipo de input que nos permite establecer
elemenos de fecha y hora en nuestras webs.
Entre los más destacables, podemos nombrar:
  • Fecha:utilizamos input type = date
  • Hora:utilizamos input type = time
  • Fecha y hora local:utilizamos input type = datetime-local
  • Mes:utilizamos input type = month
  • Semana:utilizamos input type = week

Listas de Selección

Se utilizan listas de selección, por ejemplo, cuando las casillas de verificación
se nos quedan cortas o necesitamos mostrar una lista más extensa de datos.
A través de estas listas, podemos mostrar al usuario varias opciones disponibles
para elegir una de ellas.
Entre los tipos de listas desplegables podemos destacar los siguientes:
  • Listas (Cerradas): utilizamos las etiquetas select y option
  • Listas (Abiertas): utilizamos la etiqueta datalist

Agrupación de Campos

Son etiquetas que nos permiten organizar los elementos de nuestro formulario
reagrupándolos por categorías, temáticas, o por otras clasificiones.
  • Fielset: sirve para agrupar de manera visual los campos de entrada por temática
  • Legend: son una leyenda para la etiqueta fieldset
  • Label: relacionar campo y texto de un formulario

Botones en Formularios

Podemos distinguir entre 4 tipos de botones:
  • Botón de Envio: input type = submit
  • Botón de Envio con imágen: input type = image
  • Botón de reseteo/borrado: input type = reset/li>
  • Botón sin funcionalidad: input type = button o definido
    con las etiquetas de apertura y cierre button
