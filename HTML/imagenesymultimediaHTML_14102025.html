Imágenes en HTML5

En la informática, podemos distinguir entre dos tipos de imágenes:
  • Imágenes de mapa de bits: estás están formadas por puntos de colores.
    Por ejemplo: imágenes de videojuegos de 8bits (maribros, imágenes de pixel, etc.)
  • Imágenes vectoriales: son imágenes que están formadas por formas geométricas.

Ventajas y Desventajas de Imágenes Vectoriales (SVG)

  • Ventaja:Las imágenes vectoriales, ocupan muy poco espacio.
    Además, se pueden ampliar a cualquier escala o tamaño,
    y al hacerlo, nunca se pierde la calidad de la imagen.
  • Desventaja: NO permiten representar imágenes sin
    formas definidas.
    Ejemplo: Una foto de un retrato o teléfono móvil.

Ventajas y Desventajas de Imágenes de Mapa de Bits (bmp)

  • Ventaja:Sí permiten representar imágenes sin formas definidas.
    Por ejemplo: una foto de un móvil/camara o un retrato
  • Desventaja:Las fotos si las "estiramos/ampliamos" pierden
    mucha calidad, es decir, se "pixelan", y además, son
    un tipo de imágenes que suelen ocupar mucho espacio en el
    disco duro.

Formatos de Fotos / Imágenes

  • JPG o JPEG:Son adecuadas para fotos.
    Tienen una comprensión con pérdidas pero conservan
    el color original.
  • GIF: Son adecuados para dibujos.
    No tienen pérdidas de calidad, pero
    tiene derechos de autor.
  • PNG: Son adecuados para el uso general.
    No tienen pérdidas (es decir, no pierden calidad
    al ser ampliados).
    Las fotos se convierten a este tipo de formato
    y no permiten animaciones.
    Su principal característica es que tienen transparencias.
  • WEBP: Este tipo de formato es el más adecuado para el uso
    de páginas web. Su principal característica es que tienen
    la misma calidad que las fotos JPEG/JPG junto con la calidad de comprensión de las fotos en formato PNG.
    Hoy en día, son el tipo de formato más utilizado en los Servidores web
    para representar imágenes.

Insertar imágenes en HTML5

Para poder insertar imágenes de HTML5 en una página web,
tendremos que utilizar la siguiente estructura:
  • img: La etiqueta img, es una monoetiqueta,
    que utilizaremos para poder insertar Imágenes
    en nuestra web.
  • Atributo src: El atributo src (source),
    nos ayudará a localizar la imágen que queremos
    insertar.
    Ejemplo: img src="img/foto.jpg"
  • Link de la dirección: Es la ubicación de nuestra
    imagen con el formato (extensión) que tiene.
    Ejemplo: foto.jpg, foto.jpeg, foto.png, foto.webp,
    foto.svg...


Atributos asociados a las imágenes / fotos en HTML5

  • src: indica la dirección (URI) de acceso a la imagen.
  • alt:sirve para indicar un breve texto en caso
    de que la imagen falle
  • longdesc: indica si es una URI de una descripción de
    una foto o imágen en caso de tener asociado un mapa de
    imágenes. Es decir, sirve para describir imágenes.
  • height y width:sirven para dar valor de altura y anchura
    a nuestras imágenes. Suelen venir dadas en px (pixel)
  • usemap: sirve para asociar a la imagen un mapa de
    imágenes (como un marcador)
  • ismap:sirve para definir un mapa de imágenes
  • title: a forma de tip, muestra información en un
    cuadradito amarillo de la imagen cuando pasamos el ratón
    por encima.

Figure en HTML5

La etiqueta figure en HTML5, permite asociar un texto con una imagen.
Su estructura es la siguiente:
  • figure: es la etiqueta padre. Es decir,
    el resto de etiquetas dependerán o caerán de esta.
  • img:es una etiqueta para insertar imágenes
  • figcaption: es una etiqueta que contendrá el
    texto asociado a la imagen.

Insertar vídeos de Youtube u otra plataforma.

Para poder insertar un video de Youtube u otra plataforma,
utilizaremos la etiqueta que viene asociada a estas
como puede ser iframe.

Para poder insertar un vídeo que está en local (propio pc o servidor)
utilizaremos la etiqueta video.

Para insertar un audio en una página web, utilizremos la etiqueta audio.

Atributos que podemos usar tanto en audio como en vídeo.

  • src:sirve para definir la url del audio o video
  • preload: es usado para almacenar vídeos o audios de gran
    tamaño temporalmente (buffering)
    • none: no almacena nada temporalmente
    • auto: almacena datos multimedia temporalmente
    • metadata: almacena temporalmente sólo los metadatos del archivo.
  • mediagroup: sirve para realizar una agrupación multimedia.
  • autoplay: reproduce desde el inicio.
  • loop: reproduce en bucle
  • muted:silenciado de audio
  • controls:muestra los controles del reproductor.
  • width y height:son para dar anchura y altura al vídeo
  • poster:sirve para poner poner una foto previa al vídeo.
